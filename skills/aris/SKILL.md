下列引用是从 ARIS (Auto-claude-code-research-in-sleep) 中炼化的 skills, 请在需要时加载, 可通过 Read tool 使用, 路径前缀 ${CLAUDE_PLUGIN_ROOT}. 注意: 参考其思维方式, 但不要被它的流程/模板带偏. The following references are mindsets distilled from ARIS (Auto-claude-code-research-in-sleep); Read on demand via the Read tool under ${CLAUDE_PLUGIN_ROOT}. Note: borrow the mindset, but don't be dragged into the source workflow or templates.

- skills_aris/problem-anchor: 锁定 problem 不让它在评审/讨论中漂移 + 机器人选题框架. 拿到研究方向 / 看到 scope 扩张请求 / 评审建议时用. Lock the research problem against drift + robotics problem framing. Use when receiving a research direction, evaluating scope-expansion requests, or assessing reviewer drift.
- skills_aris/literature-survey: 多源文献调研方法学. 搜文献 / mapping landscape / 找 gap 时用. Multi-source literature survey methodology. Use when searching literature, mapping a landscape, or hunting for research gaps.
- skills_aris/idea-quality: 判断 idea 是否 well-formed 和值得做的判据. 产 idea 或评估候选 idea 时用. Criteria for whether an idea is well-formed and worth pursuing. Use when generating ideas or evaluating candidates.
- skills_aris/idea-filtering: 从候选 idea 池筛出顶部的纪律. 砍 / 筛 / 评 idea 时用. Discipline for filtering candidate ideas down to a top few. Use when culling, filtering, or evaluating candidate ideas.
- skills_aris/novelty-check: 判断 idea/method 是否 novel 的判别框架. brainstorm 粗筛或定 idea 前深查时用. Framework for verifying idea/method novelty; use for quick filter during brainstorm or pre-commit deep check.
- skills_aris/empirical-pilot: 用 pilot 实验拿 empirical signal 的纪律. 投入全实验前跑 pilot 时用. Discipline for pilot experiments; use before scaling to a full experiment.
- skills_aris/method-refinement: 把 vague idea 炼成 elegant method + 公式推导——四原则/invariant object 选择/步骤分类 (identity/proposition/approximation/interpretation)/推导策略. 把 idea/approach 落成 method 或构建 coherent 理论线时用. Core principles for distilling ideas into elegant methods + formula derivation. Use when turning an idea into a concrete method or building a coherent theory line.
- skills_aris/reviewer-protocol: 跟外部 reviewer 协作 + 审稿回复——brief/review/iterate 纪律、rebuttal 三硬门安全模型、七种回复模式. 召唤 reviewer / 收 reviewer feedback / 回复审稿意见时用. Reviewer collaboration + rebuttal — brief/review/iterate discipline, rebuttal three-gate safety model, seven response modes. Use when invoking a reviewer, processing feedback, or responding to reviewer comments.
- skills_aris/experiment-planning: claim-driven 实验 roadmap. 给 stable method 设计 paper 实验时用. Claim-driven experiment roadmap. Use when designing paper experiments for a stable method.
- skills_aris/pipeline-meta: 多 phase pipeline 元规则——phase 顺序/checkpoint/预算意识/上游不稳不下游/gate before compute/知识积累/harness 优化/基金申请. 调度多 phase 时用. Meta rules for multi-phase pipelines — phase ordering, checkpoints, budget awareness, gate philosophy, knowledge accumulation, harness optimization, grant proposals. Use when orchestrating multi-phase research work.
- skills_aris/experiment-execution: 把 experiment plan 落地到运行实验 + 采集首轮结果的纪律. 实施 / 部署 / 跑实验时用. Discipline for executing an experiment plan into running experiments + initial results. Use when implementing, deploying, or running experiments.
- skills_aris/experiment-monitor: 监控运行中实验的进程状态与训练质量. 实验运行期间 / 判断是否继续时用. Monitor running experiments — process status and training quality. Use during runtime checks or when judging whether to continue.
- skills_aris/cloud-gpu: 根据任务自动选云 GPU 并控制成本的纪律. 需要按需 GPU 时用. Discipline for picking cloud GPUs by task-driven analysis with cost awareness. Use when on-demand GPU is needed.
- skills_aris/review-loop: 多轮 review→fix→re-review 改进——reviewer independence protocol、论文修复模式、restatement regression、format check 哲学. 跑 review 迭代时用. Iterative review→fix→re-review improvement — reviewer independence protocol, paper fix patterns, restatement regression, format check philosophy. Use when running a review loop.
- skills_aris/result-validation: 实验结果分析与验证——结构化发现 (Observation→Interpretation→Implication→Next step)、实验诚信六轴审计、claim 路由. 实验结束 / 准备 commit claim 时用. Result analysis and validation — structured findings, six-axis integrity audit, claim routing. Use after experiments complete or before committing to claims.
- skills_aris/paper-planning: 论文大纲规划——claims-evidence matrix、叙事结构、section-by-section 分节内容、figure 规划. 写论文前出 outline 时用. Paper outline planning — claims-evidence matrix, narrative structure, section-by-section guidance, figure planning. Use when creating a paper outline before writing.
- skills_aris/paper-writing: 论文写作准则——各节写作规范、写作质量 audit passes、参考书目规则、理论论文一致性、系统论文 blueprint. 写 LaTeX 草稿时用. Paper writing guidelines — section drafting standards, writing quality audit passes, bibliography rules, theory paper consistency, systems paper blueprint. Use when drafting LaTeX manuscript.
- skills_aris/paper-compile: LaTeX 编译与提交就绪——编译后质量信号、不同 venue 的页数规则、stale file 检测、匿名化与字体嵌入. 编译论文 / 准备提交时用. LaTeX compilation and submission readiness — post-compilation quality signals, venue-specific page count rules, stale file detection, anonymization and font embedding. Use when compiling or preparing for submission.
- skills_aris/citation-audit: 引文核查——三层审计 (存在性/元数据/上下文)、七类危险引文问题、soft-only 模式、未引用条目检测. 提交前验证引文时用. Citation audit — three-layer verification (existence/metadata/context), dangerous citation failure modes, soft-only mode, uncited entry detection. Use when verifying citations before submission.
- skills_aris/kill-argument: 对抗性审查——模拟 hostile reviewer 找出能 kill 论文的最强拒稿理由, 再由独立 adjudicator 逐点裁决. 理论论文 / 有 headline-level scope claim 时, 标准 review 后提交前用. Adversarial review — simulate a hostile reviewer to construct the single most damaging rejection argument, then adjudicate point-by-point with a second independent reviewer. Use for theory papers or papers with headline-level scope claims, after standard reviews before submission.
- skills_aris/paper-claim-audit: 零上下文论文数字核查——对比论文中每个数字与原始实验结果文件, 捕捉数字膨胀 / cherry-pick / 配置不匹配等失效模式. 论文有数字声称且原始结果文件存在时用. Zero-context paper-to-evidence audit — cross-check every paper number against raw result files, catch number inflation, cherry-pick, config mismatch. Use when paper has numeric claims and raw result files exist.
- skills_aris/proof-checker: 数学证明验证+写作——验证侧: issue 分类学/两轴严重度/侧条件清单/八轴审查/反例红队; 写作侧: feasibility triage/严格性要求/证明包结构. 论文含定理/引理/证明时用. Proof verification + writing — verification side: issue taxonomy, two-axis severity, checklists, review axes, red-team; writing side: feasibility triage, rigor requirements, proof package. Use when paper contains theorems, lemmas, or proofs.
- skills_aris/figure-styling: CVPR/ICLR/NeurIPS 学术图表视觉标准——visual/layout/arrow/配色/视觉吸引力规范 + style-ref 规则 + 审查纪律. 生成/审查论文图表时用. Academic figure visual standards — visual, layout, arrow, color palette, visual appeal specs + style-ref rules + review discipline. Use when generating or reviewing paper figures.
- skills_aris/figure-data-plots: 出版物级数据图表——图表类型选择决策树、质量检查清单、关键规则. 从实验数据生成 plots/tables 时用. Publication-quality data plots — figure type selection decision tree, quality checklist, key rules. Use when generating plots or tables from experiment data.
- skills_aris/figure-spec: 确定性 JSON→SVG 结构化图表——设计模式 (layered/hub-and-spoke/pipeline/audit cascade)、画布尺寸指南、反模式. 画架构图/流程图/审计级联图时用. Deterministic JSON→SVG structured diagrams — design patterns, canvas sizing, anti-patterns. Use for architecture, workflow, or audit cascade diagrams.
- skills_aris/figure-diagrams: 文生图——图表类型选择、架构图最佳实践、数学公式语法. 用文字描述生成 flowchart/sequence/architecture 图时用. Text-to-diagram — type selection, architecture diagram best practices, math formula syntax. Use when generating diagrams from textual descriptions.
- skills_aris/figure-description: 附图说明与引用数字——数字分配规则、多法域格式 (CN/US/EP). 为专利附图或正式技术图表写说明时用. Figure description with reference numerals — numeral assignment rules, multi-jurisdiction formats. Use when writing descriptions for patent figures or formal technical diagrams.
- skills_aris/paper-poster: 会议海报设计系统——venue 配色方案、tcbposter 布局规则、左 accent stripe 卡片设计、A0 字号规范、内容预算. 从论文生成会议海报时用. Conference poster design system — venue color schemes, layout rules, modern card design, font size rules, content budget. Use when generating a conference poster from a paper.
- skills_aris/paper-slides: 会议演讲幻灯片——按 talk 类型分 slide 模板、presentation 规则、演讲者备注结构 + pipeline 10 条硬约束. 从论文生成口头报告 slides 时用. Conference presentation slides — slide templates by talk type, presentation rules, speaker notes structure + 10 pipeline hard invariants. Use when generating oral talk slides from a paper.
- skills_aris/paper-resubmit: 论文重投——text-only microedits 硬约束、page-shrink 启发式、匿名扫描、逐轮 diff gate. 重投到新 venue 时用. Paper resubmit — hard invariants for text-only microedits, page-shrink heuristic, anonymity scan, per-round diff gate. Use when resubmitting to a different venue under tight constraints.
- skills_aris/slides-polish: 幻灯片后期视觉打磨——排版/字体/框样式修复模式、字体缩放启发式、逐页审查哲学. deck 内容锁定后打磨视觉细节时用. Post-generation slide visual polish — layout, typography, box-style fix patterns, font scaling heuristic, per-page review philosophy. Use when polishing visual details after slide content is locked.
- skills_aris/patent-principles: 专利全生命周期——新颖性/非显而易见性/prior art 检索/权利要求/说明书/实施例/审查. 专利 vs 论文管线的法律标准差异. 做专利相关工作时用. Patent lifecycle principles — novelty, non-obviousness, prior art search, claims, specification, embodiment, review. Use when doing patent-related work.
- skills_aris/dse-loop: 设计空间探索——参数推断/搜索策略 (initial sampling→directed adaptive)/停止条件. 自动调优 EDA/芯片参数时用. Design space exploration — parameter inference, search strategies, stopping conditions. Use when autonomously tuning design parameters.
- skills_aris/pixel-art: SVG 像素画——7px 网格/调色板/场景构图/气泡与箭头机制. 画像素插图时用. SVG pixel art — grid system, color palette, scene composition, bubble and arrow mechanics. Use when creating pixel art illustrations.
